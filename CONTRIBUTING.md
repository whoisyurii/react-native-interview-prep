# Contributing

Thanks for improving this React Native interview guide.

## Content Rules

- Prefer questions that expose judgment, debugging ability, or production experience.
- Keep answers concise: a few sentences, not a tutorial.
- Use current React Native facts. The default baseline is React Native 0.82+ and the New Architecture-only era.
- Avoid old bridge-era trivia unless the question is explicitly about migration or compatibility.
- Add a source link when a claim is version-sensitive.

## Question Format

- **Answer:** Short answer with a concrete caveat or tradeoff.

## Level Guide

- **Junior**: day-to-day app development, UI state, lists, effects, accessibility, debugging basics.
- **Middle**: performance, native boundaries, New Architecture concepts, release safety, dependencies.
- **Senior**: architecture decisions, migrations, observability, team process, production tradeoffs.
- **God Mode**: renderer internals, JSI, threading, native module design, engines, platform strategy.
